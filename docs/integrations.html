Integrations

The tools wired together to make the course project run. Each one chosen for a specific job, not because it was on a list.

01

Data layer

Pg
PostgreSQL
Primary database

Relational. The reference implementation against which the course concepts get tested.

Re
Redis
Cache layer

Key-value store for session state and hot-path query caching. A practical contrast to Postgres.

Mo
MongoDB
Document store

For the parts of the project that are genuinely document-shaped. Used sparingly, by design.

02

Application layer

Rx
React + TS
Frontend

Vite-powered build, Tailwind for utility CSS, Radix primitives where it matters.

No
Node.js
API server

Express-style routing, parameterized queries, never string concatenation anywhere near SQL.

Py
Python
Analytics & scripts

Pandas for data exploration, ad-hoc scripts for seeding and migrations.

03

Ops & infra

Gh
GitHub Actions
CI / CD

Every push runs tests, every merge to main deploys. Confidence comes from automation.

Cf
Cloudflare
DNS & SSL

DNS, edge caching, automatic certificates. The boring infrastructure that should just work.

Vp
RackNerd VPS
Host

Ubuntu 22.04, Nginx in front, self-managed. The course project lives on a real server.

Ob
Obsidian
Notes

Where the messy thinking happens before any of it makes its way into this wiki.
