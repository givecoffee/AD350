What ACID Actually Promises (and What It Doesn't)

Most explanations of ACID list the four words (Atomicity, Consistency, Isolation, Durability) and stop there. The interesting part is understanding what each guarantee provides and, more importantly, what it doesn't.

Atomicity ensures "all or nothing" operations, but it doesn't guarantee that concurrent transactions won't interfere with each other. That's where isolation levels come in.

Indexes as a Contract with the Query Planner

An index isn't just a performance setting—it's a promise to the query planner about how data will be accessed. When you create an index, you're telling PostgreSQL: "I expect to query this column often, and here's how I'll use it."

The query planner uses these promises to build efficient execution plans. But indexes come with costs: storage overhead and write performance degradation.
